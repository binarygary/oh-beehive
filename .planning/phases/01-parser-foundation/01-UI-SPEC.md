## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps (`gap-1`), tight inline spacing |
| sm | 8px | Compact element spacing (`gap-2`, `space-y-2`, `p-2`) |
| md | 16px | Default element spacing (`gap-4`, `space-y-4`) |
| xl | 24px | Page padding mobile (`p-6`), card body padding (`p-6`), section gaps (`space-y-6`) |
| 2xl | 32px | Page padding desktop (`lg:p-8`) |
| sidebar | 256px | Sidebar width (`w-64`) ‚Äî fixed layout token |

**Pre-existing undeclared values (not governed by this spec):** `space-y-1.5` (6px) and `gap-1.5` (6px) appear in `create.blade.php` and `navigation.blade.php` as inherited patterns (label-to-input gaps, icon-to-text pairs in nav links). These are not introduced in Phase 1 and are not part of the declared scale. Do not use these values in new Phase 1 code; use `sm` (8px) as the nearest governed token.

**Note on `p-5` (20px):** The existing codebase uses `p-5` as a DaisyUI card body default. This value is pre-existing and not introduced in Phase 1. It is not part of the declared scale. New Phase 1 markup uses `p-6` (`xl`, 24px) for card body padding.

**Source:** Pattern-matched from `create.blade.php`, `navigation.blade.php`, `app.blade.php`.

## Typography

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Body | 14px (`text-sm`) | 400 (`font-normal`) | 1.5 (`leading-relaxed` on textarea) |
| Label | 14px (`text-sm`) | 400 (`font-normal`) | 1.5 |
| Heading (section) | 12px (`text-xs`) | 600 (`font-semibold`) | 1 ‚Äî uppercase + tracking-wide (`uppercase tracking-wide`) |
| Heading (page) | 24px (`text-2xl`) | 600 (`font-semibold`) | 1.2 |

**Weight consolidation:** This spec declares exactly 2 weights. `font-medium` (500) is consolidated into `font-normal` (400) ‚Äî the visual difference at 14px on Figtree is negligible. `font-bold` (700) is consolidated into `font-semibold` (600) ‚Äî page headings use `font-semibold` instead. Existing files that use `font-medium` or `font-bold` are pre-existing; do not touch them in Phase 1 beyond the two files being modified (`create.blade.php`, `edit.blade.php`). All new markup introduced in Phase 1 must use only `font-normal` or `font-semibold`.

Note: The project uses exactly these four roles. No display size is used in Phase 1 scope. Section headings are intentionally small-caps style (`text-xs font-semibold uppercase tracking-wide text-base-content/60`).

**Source:** Observed across `create.blade.php` (`h1`, `h2`, form labels, textarea), `navigation.blade.php`.

## Color

All colors use DaisyUI v5 semantic tokens resolved by the `bumblebee` theme. Do not use raw hex values.

| Role | Token | Usage |
|------|-------|-------|
| Dominant (60%) | `bg-base-200` | Main content area background (drawer-content) |
| Secondary (30%) | `bg-base-100` | Sidebar, cards (`card bg-base-100`), navbar, dropdowns |
| Border | `border-base-300` | All card borders, sidebar border, navbar border |
| Accent (10%) | `primary` / `bg-primary` | Brand icon background, active nav `btn-primary` (selected score buttons, checkbox `checkbox-primary`), primary submit button |
| Destructive | `error` | Required field asterisks (`text-error`), validation error text, Log Out button text; never used for primary actions |
| Warning | `warning` | `alert alert-warning` ‚Äî parse error notice and existing follow-up questions alert |
| Success | `success` | `btn-success` ‚Äî yes/positive boolean toggle buttons (queen seen, eggs, larvae, feeding done) |
| Muted | `text-base-content/40` | Helper text, section heading opacity, user email, chevron opacity |
| Semi-muted | `text-base-content/50` | Back link, analyzing spinner text |

Accent (`primary`) is reserved for:
- Parse button (new in Phase 1 ‚Äî `btn btn-primary btn-sm`)
- Selected score step buttons (`btn-primary` active state in `join`)
- Disease checkboxes (`checkbox-primary`)
- Primary submit button (`btn-primary`)
- Brand avatar / logo background (`bg-primary`)
- Active nav link state (DaisyUI `active` class in `menu`)

**Source:** `app.blade.php` (data-theme, bg-base-200), `create.blade.php` (card colors, button variants, error/warning/success), `navigation.blade.php` (sidebar bg-base-100, border-base-300).

## Phase 1 UI Changes

Phase 1 is infrastructure-focused. The only two UI surface changes are:

### 1. Parse Button

**Location:** Directly below the raw notes textarea, inside the existing `card-body` div in `create.blade.php` and `edit.blade.php`.

**Markup contract:**

```blade
<button type="button" wire:click="parse" wire:loading.attr="disabled" wire:target="parse"
        class="btn btn-primary btn-sm">
    <span wire:loading.remove wire:target="parse">Parse Notes</span>
    <span wire:loading wire:target="parse" class="flex items-center gap-2">
        <span class="loading loading-spinner loading-xs"></span>
        Analyzing‚Ä¶
    </span>
</button>
```

**Rationale:** `btn-primary` is correct ‚Äî this is the primary action within the notes card, and the existing "Save Inspection" button at the bottom of the form is also `btn-primary`. Parse is the primary action for the notes section. The two buttons are in different visual zones (inside the card vs. the form footer) so there is no ambiguity.

**Position within card:** Below the textarea, above any error alert. Order: `<textarea>` ‚Üí `<button type="button">Parse Notes</button>` ‚Üí `[alert-warning if parseError]` ‚Üí `@error('rawNotes')`.

**Loading state:** The existing `wire:loading` / `wire:loading.remove` pattern used throughout the form. The "Analyzing‚Ä¶" spinner that previously appeared in the card header (`wire:target="updatedRawNotes"`) is relocated to inside this button. The header `span wire:loading` for the old debounce target is removed. Gap between spinner and text uses `gap-2` (8px / `sm` token) ‚Äî governed value.

**Disabled state:** Button is disabled (`btn-disabled`) while `strlen(trim($rawNotes)) < 15`. Disabled state uses DaisyUI `btn-disabled` class rather than native `disabled` attribute to preserve Livewire wire binding.

### 2. Parse Error Alert

**Location:** Immediately below the Parse button, inside the `card-body`, only when a parse error has occurred. Disappears on next successful parse.

**Markup contract:**

```blade
@if($parseError)
<div class="alert alert-warning">
    <svg xmlns="http://www.w3.org/2000/svg" class="h-5 w-5 shrink-0" fill="none"
         viewBox="0 0 24 24" stroke="currentColor" stroke-width="2">
        <path stroke-linecap="round" stroke-linejoin="round"
              d="M12 9v2m0 4h.01m-6.938 4h13.856c1.54 0 2.502-1.667 1.732-3L13.732 4c-.77-1.333-2.694-1.333-3.464 0L3.34 16c-.77 1.333.192 3 1.732 3z" />
    </svg>
    <span>Could not parse notes. You can fill fields manually below.</span>
</div>
@endif
```

**Icon:** Warning triangle (Heroicons `exclamation` solid path, stroke-based variant) ‚Äî consistent with existing alert-warning usage in follow-up questions panel which uses a question-circle icon. The triangle icon more clearly signals failure vs. question.

**Behavior:**
- Non-blocking: structured fields remain visible and editable when alert is shown.
- Auto-clears: `$parseError` property set to `false` on next successful parse call.
- No dismiss button. Keeper's natural action (fix notes, click Parse again) clears it.

**Source of decisions:** CONTEXT.md D-04, D-05, D-06.

## Interaction States

The following states must be implemented for the two new UI elements:

### Parse Button States

| State | Condition | Visual |
|-------|-----------|--------|
| Default | `strlen(trim($rawNotes)) >= 15` | `btn btn-primary btn-sm` ‚Äî amber fill (bumblebee primary) |
| Disabled | `strlen(trim($rawNotes)) < 15` | `btn btn-primary btn-sm btn-disabled` ‚Äî dimmed, no pointer events |
| Loading | `wire:loading` targeting `parse` method | Spinner + "Analyzing‚Ä¶" text inside button; button disabled during call |

### Parse Error Alert States

| State | Condition | Visual |
|-------|-----------|--------|
| Hidden | `$parseError === false` | Not rendered (`@if($parseError)` gate) |
| Visible | `$parseError === true` | `alert alert-warning` with warning triangle icon and error copy |

### Fields Panel Loading State

| State | Condition | Visual |
|-------|-----------|--------|
| Active | parse in progress | `wire:loading.class="opacity-40 pointer-events-none"` on fields wrapper ‚Äî unchanged from existing pattern |
