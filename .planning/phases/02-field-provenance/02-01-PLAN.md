## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| keeper input → Livewire component state | Untrusted note text and manual field edits drive parse-time and update-hook state changes |
| parser response → Blade rendering | External parser output determines which fields receive provenance labels |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-01 | T | `resources/views/livewire/pages/inspections/create.blade.php` parse assignment block | mitigate | Only mark provenance for explicit known parser keys already mapped in the component, per D-02 and D-09; do not render arbitrary parser keys. |
| T-02-02 | I | create-form provenance labels | mitigate | Keep provenance ephemeral in component state only per D-01 and never persist it to `Inspection` records or expose it outside the current form render. |
| T-02-03 | R | manual override detection | mitigate | Use Livewire update hooks plus eager `wire:model.blur` / `.change` bindings so the rendered provenance state matches the keeper’s latest action and cannot falsely imply the AI still owns a field. |
</threat_model>
