<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| saved inspection data → Livewire component state | Existing persisted values are loaded into the edit form before any new parse occurs |
| keeper input + parser response → edit-form render | Manual edits and fresh parser output compete to determine which fields appear AI-owned |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-04 | T | `resources/views/livewire/pages/inspections/edit.blade.php` parse assignment block | mitigate | Restrict provenance updates to the explicit parser-to-property assignment list so unexpected parser keys cannot affect rendered edit-form badges. |
| T-02-05 | I | edit-form provenance state | mitigate | Keep `$aiFilledFields` transient and separate from the persisted `Inspection` model so parser ownership is never stored or leaked beyond the active edit session. |
| T-02-06 | R | edit-form manual override flow | mitigate | Clear provenance only after actual keeper-driven property changes using Livewire hooks and eager bindings, so the UI cannot falsely attribute an edited saved value to AI after override. |
</threat_model>
