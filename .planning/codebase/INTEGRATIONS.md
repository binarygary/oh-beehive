# External Integrations

**Analysis Date:** 2026-04-04

## APIs & External Services

**AI Parsing:**
- OpenAI (GPT-4o-mini model)
  - What it's used for: Parsing raw inspection notes into structured fields via `InspectionParserService`
  - SDK/Client: `openai-php/client` v0.19.1+
  - Auth: `OPENAI_API_KEY` environment variable
  - Config file: `config/services.php` under `'openai'` key
  - Usage: `app/Services/InspectionParserService.php` performs chat completions with JSON response format

**Optional Email Services:**
- Postmark - Email delivery (configured but not required)
  - Auth: `POSTMARK_API_KEY`
  - Config: `config/services.php`
- Resend - Email delivery (configured but not required)
  - Auth: `RESEND_API_KEY`
  - Config: `config/services.php`
- AWS SES - Email delivery via AWS (configured but not required)
  - Auth: `AWS_ACCESS_KEY_ID`, `AWS_SECRET_ACCESS_KEY`
  - Config: `config/services.php`

**Optional Notifications:**
- Slack - Bot notifications (configured but not required)
  - Auth: `SLACK_BOT_USER_OAUTH_TOKEN`
  - Channel: `SLACK_BOT_USER_DEFAULT_CHANNEL`
  - Config: `config/services.php`

## Data Storage

**Databases:**
- SQLite (default)
  - Connection: `DB_CONNECTION=sqlite`
  - Database file: `database/database.sqlite`
  - Client: Laravel Eloquent ORM (built-in)
  - Used for: All domain data (Users, Hives, Inspections)
  - Features: Foreign key constraints enabled by default

- MySQL (optional, configured)
  - Connection string: `DB_*` environment variables
  - Not currently used but framework supports it

**File Storage:**
- Local filesystem only
  - Default disk: `FILESYSTEM_DISK=local` (private storage)
  - Public disk available at `storage/app/public`
  - S3 configuration present in `config/filesystems.php` but not required

**Caching:**
- Database cache store (default)
  - Config: `config/cache.php` - `CACHE_STORE=database`
  - Table: Auto-created by Laravel
  - Alternative stores configured but not active: Redis, Memcached, DynamoDB
  - Prefix: Auto-generated from app name

**Session Storage:**
- Database-backed sessions (default)
  - Driver: `SESSION_DRIVER=database`
  - Table: `sessions`
  - Lifetime: `SESSION_LIFETIME=120` minutes
  - Encryption: `SESSION_ENCRYPT=false`

**Queue Storage:**
- Database queue driver (default)
  - Config: `QUEUE_CONNECTION=database`
  - Table: `jobs` (failed jobs in `failed_jobs`)
  - Alternative drivers configured but not active: Redis, Beanstalkd, AWS SQS
  - Sync driver available for testing

## Authentication & Identity

**Auth Provider:**
- Custom (Laravel built-in)
  - Implementation: Session-based with Eloquent user provider
  - Guard: `web` (session-based)
  - User model: `App\Models\User`
  - Password hashing: Bcrypt with `BCRYPT_ROUNDS=12`
  - Password reset tokens: Database-backed with 60-minute expiry
  - Password confirmation timeout: 3 hours
  - No third-party OAuth (custom user creation via `php artisan make:user`)
  - Multi-tenant: All data (hives, inspections) scoped to authenticated user via `user_id` FK

## Monitoring & Observability

**Error Tracking:**
- None detected - Framework default error handling

**Logs:**
- File-based logging
  - Config: `LOG_CHANNEL=stack` with `LOG_STACK=single`
  - Level: `LOG_LEVEL=debug` (development default)
  - Sent to: `storage/logs/`
  - Alternative: Laravel Pail for streaming logs during development

**Mail Logging:**
- Mail driver: `MAIL_MAILER=log` (development)
  - Mails logged instead of sent
  - Channel: `LOG_LOG_CHANNEL` (optional)
  - For production: Configure to Postmark, Resend, SES, or SMTP

## Environment Configuration

**Required env vars:**
- `APP_KEY` - Generated by `php artisan key:generate`
- `APP_URL` - Application URL for session and redirects
- `DB_CONNECTION` - Database driver (default: `sqlite`)
- `OPENAI_API_KEY` - Required for inspection parsing feature

**Optional env vars (with defaults):**
- `APP_NAME`, `APP_ENV`, `APP_DEBUG`, `APP_LOCALE`
- `BCRYPT_ROUNDS` - Password hashing iterations (default: 12)
- `LOG_CHANNEL`, `LOG_LEVEL` - Logging configuration
- `SESSION_LIFETIME` - Minutes (default: 120)
- `CACHE_STORE` - Backend store (default: database)
- `QUEUE_CONNECTION` - Job queue driver (default: database)
- `MAIL_MAILER` - Email service (default: log)
- `REDIS_*` - Redis connection (optional, not required)
- `AWS_*` - AWS credentials for S3/SES (optional)
- `POSTMARK_API_KEY`, `RESEND_API_KEY`, `SLACK_*` - Optional external services

**Secrets location:**
- `.env` file (local development only, not committed)
- Environment variables (production deployment)
- Secrets handled by deployment platform (e.g., GitHub Secrets, Heroku Config Vars, etc.)

**Setup script:**
- `composer setup` orchestrates:
  1. `composer install`
  2. Copy `.env.example` to `.env` (if missing)
  3. Generate `APP_KEY`
  4. Run migrations
  5. Install npm packages
  6. Build frontend assets with Vite
